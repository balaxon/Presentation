Lodash
What is Lodash?
Lodash is a JavaScript library which provides utility functions for common programming tasks using the functional programming paradigm.
Why Lodash?
Lodash makes JavaScript easier by taking the hassle out of working with arrays, numbers, objects, strings, etc.

Lodash is a JavaScript library that helps programmers write more concise and maintainable JavaScript.
It can be broken down into several main areas:
Utilities:
For simplifying common programming tasks such as determining type as well as simplifying math operations.
For exaple:
  • times
  • range
Times:
Invokes the iteratee n times, returning an array of the results of each invocation. The iteratee is invoked with one argument; (index).

Example:
Range:
Creates an array of numbers (positive and/or negative) progressing from start up to, but not including, end. A step of -1 is used if a negative start is specified without an end or step. If end is not specified, it's set to start with start then set to 0.

Example:
Function:
Simplifying binding, decorating, constraining, throttling, debouncing, currying, and changing the pointer.
For exaple:
  • ary
  • bind
Ary:
Creates a function that invokes func, with up to n arguments, ignoring any additional arguments.

Example:
Bind:
Creates a function that invokes func with the this binding of thisArg and partials prepended to the arguments it receives.

Example:
String:
Conversion functions for performing basic string operations, such as trimming, converting to uppercase, camel case, etc.
For exaple:
  • camelCase
  • lowerCase
CamelCase:
Converts string to camel case.

Example:
LowerCase:
Converts string, as space separated words, to lower case.

Array:
Creating, splitting, combining, modifying, and compressing
For exaple:
  • compact
  • drop
Compact:
Creates an array with all falsey values removed. The values false, null, 0, "", undefined, and NaN are falsey.

Example:
Drop:
Creates a slice of array with n elements dropped from the beginning.

Example:
Collection:
Iterating, sorting, filtering, splitting, and building.
For exaple:
  • countBy
  • filter
CountBy:
Creates an object composed of keys generated from the results of running each element of collection thru iteratee. The corresponding value of each key is the number of times the key was returned by iteratee. The iteratee is invoked with one argument: (value).

Example:
Filter:
Iterates over elements of collection, returning an array of all elements predicate returns truthy for. The predicate is invoked with three arguments: (value, index|key, collection).

Example:
Object:
Accessing, extending, merging, defaults, and transforming.
For exaple:
  • assign
  • create
Assign:
Assigns own enumerable string keyed properties of source objects to the destination object. Source objects are applied from left to right. Subsequent sources overwrite property assignments of previous sources.

Example:
Create:
Creates an object that inherits from the prototype object. If a properties object is given, its own enumerable string keyed properties are assigned to the created object.

Example:
Seq:
Chaining, wrapping, filtering, and testing.
For exaple:
  • chain
  • tap
Chain:
Creates a lodash wrapper instance that wraps value with explicit method chain sequences enabled. The result of such sequences must be unwrapped with _#value.

Example:
Tap:
This method invokes interceptor and returns value. The interceptor is invoked with one argument; (value). The purpose of this method is to "tap into" a method chain sequence in order to modify intermediate results.
